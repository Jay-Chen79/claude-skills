# PDCA常见主题参考数据

各类主题的基线值、原因分析、措施参考。

## 时间类改进主题

### 缩短急诊预检分诊时间

**指标定义**：从患者到达急诊至完成分诊的时间（分钟）

**基线参考**：8-15分钟
**目标参考**：5-8分钟
**改进幅度**：30-40%

**5Why分析示例**：
```
问题：急诊分诊时间过长（平均12分钟）

Why1: 为什么分诊时间长？
  → 患者排队等候时间长

Why2: 为什么排队等候时间长？
  → 分诊台人手不足，同时只能处理1-2人

Why3: 为什么同时只能处理1-2人？
  → 信息录入耗时，需要手工登记多项内容

Why4: 为什么信息录入耗时？
  → 没有预检分诊信息化系统，全靠手写

Why5: 为什么没有信息化系统？
  → 原有系统老旧，未升级改造 ★根本原因
```

**常见改进措施**：
1. 优化分诊流程，简化非必要登记项
2. 引入预检分诊信息系统
3. 增设分诊通道，高峰时段双人分诊
4. 建立快速分诊通道（危重患者）
5. 培训提高分诊效率

---

### 缩短检验报告TAT时间

**指标定义**：从标本接收至报告发出的时间

**基线参考**：
- 常规生化：3-5小时
- 急诊生化：45-90分钟
- 血常规：30-50分钟

**目标参考**：缩短25-35%

**原因分析（4M）**：
- 人：人员配置不足、操作不熟练
- 机：仪器老旧、故障率高、维护不及时
- 料：试剂准备不充分、标本问题多
- 法：流程繁琐、审核环节多、信息传递慢

**常见改进措施**：
1. 优化标本处理流程
2. 调整仪器运行安排
3. 简化审核流程（双签改单签）
4. LIS系统自动传输
5. 增加高峰时段人员

---

### 缩短出院结算等候时间

**指标定义**：患者从到达收费窗口至完成结算的时间

**基线参考**：30-60分钟
**目标参考**：15-25分钟
**改进幅度**：40-50%

**原因分析**：
- 费用核对耗时（医保、自费分别核算）
- 窗口数量有限
- 集中时段排队
- 材料准备不全需往返

**常见改进措施**：
1. 推行床旁结算/移动结算
2. 预审核制度（提前一天核对费用）
3. 增设自助结算机
4. 优化窗口排班
5. 出院准备清单提前告知

---

## 数据波动参考

生成虚拟数据时，加入适度波动：

**周度数据波动**：±10-15%
**日度数据波动**：±15-25%

**波动代码示例**：
```python
import numpy as np

def generate_trend_data(baseline, target, weeks_before, weeks_after):
    """生成有波动的趋势数据"""
    # 改进前数据（在基线附近波动）
    before = baseline + np.random.normal(0, baseline*0.1, weeks_before)
    
    # 改进过程（逐渐下降/上升）
    transition = np.linspace(baseline, target, 3)
    transition += np.random.normal(0, baseline*0.08, 3)
    
    # 改进后数据（在目标附近波动）
    after = target + np.random.normal(0, target*0.1, weeks_after)
    
    return list(before) + list(transition) + list(after)
```
