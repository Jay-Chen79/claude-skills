# 医院PDCA循环项目生成器

为质量管理培训和等级医院评审生成全套PDCA项目虚拟资料。

## PDCA四阶段概述

```
    ┌─────────┐
    │ P 计划  │ → 分析问题、找原因、定目标、做计划
    └────┬────┘
         │
         ▼
    ┌─────────┐
    │ D 执行  │ → 实施改进措施、跟踪进度
    └────┬────┘
         │
         ▼
    ┌─────────┐
    │ C 检查  │ → 收集效果数据、对比分析
    └────┬────┘
         │
         ▼
    ┌─────────┐
    │ A 处理  │ → 标准化成功经验、启动下一循环
    └─────────┘
```

## Step 1: 需求收集

收到PDCA主题后，询问以下信息：

### 必问信息
- **科室/部门**：临床科室/医技科室/行政后勤/护理单元
- **项目周期**：1-3个月（PDCA特点是短平快）
- **指标类型**：
  - 时间指标：等候时间、周转时间、响应时间
  - 比率指标：合格率、完成率、准确率
  - 计数指标：差错数、投诉数、事件数
- **改进方向**：提高/降低/缩短

### 可选信息（有合理默认值）
- 参与人员（默认3-5人小组）
- 数据收集频次（默认每周）
- 预期改进幅度（默认20-40%）

### 问询策略
- 一次问3-4个核心问题
- 简单项目可快速推进
- 确认关键参数后开始生成

## Step 2: 数据设计原则

### 改进幅度参考

PDCA周期短，改进幅度通常比品管圈小：

| 改进程度 | 幅度范围 | 说明 |
|----------|----------|------|
| 一般 | 15-25% | 常规改进 |
| 良好 | 25-40% | 较好效果 |
| 优秀 | 40-50% | 优秀案例 |

**注意**：不超过50%，保持现实性

### 常见指标基线参考

**时间类**：
| 指标 | 典型基线 | 合理改进后 |
|------|----------|------------|
| 急诊分诊时间 | 8-15分钟 | 5-8分钟 |
| 检验报告TAT | 3-5小时 | 2-3小时 |
| 出院结算时间 | 40-60分钟 | 20-35分钟 |
| 床旁交接时间 | 15-25分钟 | 10-15分钟 |

**比率类**：
| 指标 | 典型基线 | 合理改进后 |
|------|----------|------------|
| 医嘱执行准确率 | 92-96% | 97-99% |
| 手卫生依从率 | 65-75% | 80-90% |
| 病历完整率 | 85-92% | 95-98% |
| 预约准时率 | 70-80% | 85-92% |

**计数类**（月度）：
| 指标 | 典型基线 | 合理改进后 |
|------|----------|------------|
| 给药差错数 | 8-15次 | 3-6次 |
| 患者投诉数 | 10-20次 | 5-10次 |
| 标本退回数 | 20-40次 | 10-20次 |

### 数据逻辑要求

1. **趋势合理**：改进不是突变，应有渐进过程
2. **波动适度**：数据有正常波动，非完美直线
3. **前后呼应**：原因分析→措施→效果逻辑链完整

## Step 4: 各阶段详细生成指南

### P阶段（Plan 计划）

#### 4.1.1 问题陈述与背景

**结构**：
```
一、问题描述
   - 什么问题？
   - 发生在哪里？
   - 影响是什么？

二、背景分析
   - 相关政策/标准要求
   - 科室现状
   - 改进的必要性和紧迫性

三、项目范围
   - 涉及科室/人员
   - 时间范围
   - 不包括的内容
```

#### 4.1.2 现状数据收集

**Excel表结构**：
```
| 日期 | 数据项1 | 数据项2 | ... | 备注 |
|------|---------|---------|-----|------|
| 第1周 | xx | xx | ... | |
| 第2周 | xx | xx | ... | |
...
| 汇总 | =AVERAGE() | =AVERAGE() | | |
```

**收集周期**：通常2-4周基线数据

#### 4.1.3 原因分析

**5Why分析法模板**：
```
问题：[具体问题描述]

为什么1？→ [直接原因]
  为什么2？→ [深层原因]
    为什么3？→ [更深原因]
      为什么4？→ [根本原因接近]
        为什么5？→ [根本原因] ★
```

**简化鱼骨图**（4M）：
- 人（Man）：人员因素
- 机（Machine）：设备/系统因素
- 料（Material）：物资/信息因素
- 法（Method）：流程/方法因素

#### 4.1.4 目标设定

**SMART原则**：
- S（Specific）：具体明确
- M（Measurable）：可测量
- A（Achievable）：可实现
- R（Relevant）：相关性
- T（Time-bound）：有时限

**目标陈述模板**：
```
在[时间范围]内，将[指标名称]从[现状值]改进至[目标值]，
改进幅度达到[X]%。
```

#### 4.1.5 PDCA计划表

**Excel格式**：
```
| 阶段 | 序号 | 工作内容 | 负责人 | 开始时间 | 完成时间 | 状态 |
|------|------|----------|--------|----------|----------|------|
| P | 1 | 收集现状数据 | 张三 | W1 | W2 | ✓ |
| P | 2 | 分析原因 | 李四 | W2 | W3 | ✓ |
| D | 3 | 实施措施1 | 王五 | W3 | W5 | ✓ |
| D | 4 | 实施措施2 | 张三 | W3 | W6 | ✓ |
| C | 5 | 收集效果数据 | 李四 | W6 | W8 | ✓ |
| C | 6 | 效果分析 | 全体 | W8 | W9 | ✓ |
| A | 7 | 标准化 | 王五 | W9 | W10 | ✓ |
| A | 8 | 总结报告 | 张三 | W10 | W11 | ✓ |
```

### D阶段（Do 执行）

#### 4.2.1 改进措施清单

**Excel格式**：
```
| 序号 | 针对原因 | 改进措施 | 负责人 | 资源需求 | 完成时限 | 验收标准 |
|------|----------|----------|--------|----------|----------|----------|
| 1 | 原因A | 措施描述 | 张三 | 无/少量 | W4 | 具体标准 |
| 2 | 原因B | 措施描述 | 李四 | 设备X | W5 | 具体标准 |
```

**措施设计原则**：
- 针对性：每个措施对应具体原因
- 可操作：具体可执行，非空话
- 资源可及：不需要大量额外资源
- 时限明确：有具体完成时间

#### 4.2.2 进度跟踪表

**Excel格式**：
```
| 措施 | 计划进度 | W1 | W2 | W3 | W4 | W5 | W6 | 实际完成 | 偏差说明 |
|------|----------|----|----|----|----|----|----|----------|----------|
| 措施1 | W1-W3 | ▓ | ▓ | ▓ | | | | W3 | 按时 |
| 措施2 | W2-W5 | | ▓ | ▓ | ▓ | ▓ | | W6 | 延迟1周 |
```

#### 4.2.3 执行过程记录

**内容要点**：
- 各措施实施情况
- 遇到的困难及解决方法
- 过程中的调整
- 阶段性观察

### C阶段（Check 检查）

#### 4.3.1 改进后数据

**与改进前数据结构一致**，便于对比

收集周期：通常2-4周效果数据

#### 4.3.2 前后对比图表

**必须包含**：
- 柱状图或折线图对比
- 改进前后均值标注
- 目标线标注
- 改进幅度计算

**图表代码参考**：
```python
import matplotlib.pyplot as plt
import numpy as np

def create_comparison_chart(before_data, after_data, target, 
                           title, ylabel, output_path):
    """改进前后对比图"""
    fig, ax = plt.subplots(figsize=(10, 6))
    
    # 数据准备
    x = np.arange(len(before_data) + len(after_data))
    all_data = list(before_data) + list(after_data)
    colors = ['#ff7f0e'] * len(before_data) + ['#1f77b4'] * len(after_data)
    
    # 柱状图
    bars = ax.bar(x, all_data, color=colors, edgecolor='black')
    
    # 均值线
    before_mean = np.mean(before_data)
    after_mean = np.mean(after_data)
    ax.axhline(y=before_mean, color='#ff7f0e', linestyle='--', 
               label=f'改进前均值: {before_mean:.1f}')
    ax.axhline(y=after_mean, color='#1f77b4', linestyle='--',
               label=f'改进后均值: {after_mean:.1f}')
    
    # 目标线
    ax.axhline(y=target, color='red', linestyle='-', linewidth=2,
               label=f'目标值: {target}')
    
    # 分隔线
    ax.axvline(x=len(before_data)-0.5, color='gray', linestyle=':')
    
    ax.set_ylabel(ylabel)
    ax.set_title(title)
    ax.legend()
    
    plt.tight_layout()
    plt.savefig(output_path, dpi=300)
    plt.close()
```

#### 4.3.3 目标达成分析

**计算公式**：

```
改进幅度 = (改进前值 - 改进后值) / 改进前值 × 100%  [负向指标]
改进幅度 = (改进后值 - 改进前值) / 改进前值 × 100%  [正向指标]

目标达成率 = 实际改进幅度 / 计划改进幅度 × 100%
```

#### 4.3.4 效果评价报告

**结构**：
```
一、数据对比
   - 改进前数据汇总
   - 改进后数据汇总
   - 统计检验结果（如适用）

二、目标达成情况
   - 目标值 vs 实际值
   - 达成率计算

三、效果评价
   - 达成/部分达成/未达成
   - 原因分析（如未完全达成）

四、其他收获
   - 意外发现
   - 附带改进
```
