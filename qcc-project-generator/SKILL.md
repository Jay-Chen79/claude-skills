# 医院品管圈（QCC）项目生成器

为品管圈培训教学生成全套虚拟项目资料。

## 品管圈十大步骤

1. **主题选定** - 选题理由、评价矩阵
2. **活动计划拟定** - 甘特图、圈员分工
3. **现状把握** - 流程图、查检表、柏拉图
4. **目标设定** - 目标值计算、衡量指标
5. **解析** - 鱼骨图、真因验证
6. **对策拟定** - 对策评价、实施计划
7. **对策实施与检讨** - 执行记录、调整
8. **效果确认** - 改善前后对比、统计检验
9. **标准化** - SOP、流程图、培训
10. **检讨与改进** - 无形成果、后续计划

## Step 1: 需求收集

收到品管圈主题后，询问以下信息：

### 必问信息
- **科室类型**：内科/外科/急诊/ICU/手术室/门诊/医技/护理单元等
- **圈员组成**：人数（通常5-10人）、职称结构
- **活动周期**：通常6-12个月
- **指标类型**：
  - 正向指标（越高越好）：如知晓率、合格率
  - 负向指标（越低越好）：如感染率、跌倒率、差错率

### 可选信息（有默认值）
- 圈名、圈徽（可自动生成创意名称）
- 辅导员信息
- 医院等级（三甲/三乙/二甲等）

### 问询策略
- 一次问3-4个核心问题
- 根据主题智能推断合理默认值
- 确认关键参数后开始生成

## Step 2: 数据设计原则

### 现实性约束

**改善幅度参考**（基于真实QCC案例）：
- 一般改善：30-50%
- 良好改善：50-70%
- 优秀改善：70-85%
- 不要超过90%（不现实）

**常见医疗指标基线参考**：
| 指标 | 典型基线 | 合理改善后 |
|------|----------|------------|
| 跌倒发生率 | 0.3-0.8‰ | 0.1-0.3‰ |
| 手卫生依从率 | 60-75% | 85-95% |
| 给药差错率 | 0.1-0.5% | 0.02-0.1% |
| 标本合格率 | 85-92% | 95-98% |
| 患者满意度 | 80-88% | 92-96% |
| 交接班完整率 | 70-80% | 90-95% |

### 数据逻辑自洽

1. **柏拉图数据** → 累计百分比须达80%确定主要原因
2. **真因验证** → 验证结果须支持鱼骨图分析
3. **对策** → 须针对真因制定
4. **效果数据** → 改善趋势须合理（非突变）

### 时间线一致性

- 活动计划时间 = 实际数据收集时间
- 改善前数据期 < 对策实施期 < 改善后数据期
- 各阶段时长符合甘特图安排

## Step 4: 各步骤详细生成指南

### 4.1 主题选定评价表

**评价维度**（5-7项，每项1-5分）：
- 上级政策（重要性）
- 可行性
- 迫切性
- 圈能力
- 患者安全相关性
- 成本效益

**生成要求**：
- 列出3-5个候选主题
- 所选主题得分最高
- 第二名分数接近但略低（差2-5分）

### 4.2 活动计划甘特图

**标准格式**：
- 横轴：周次或月份
- 纵轴：十大步骤 + 细分任务
- 标注：计划线（实线）、实际线（虚线）
- 责任人分配

**时间分配参考**（以6个月为例）：
| 步骤 | 建议时长 |
|------|----------|
| 主题选定 | 2周 |
| 计划拟定 | 1周 |
| 现状把握 | 4-6周 |
| 目标设定 | 1周 |
| 解析 | 3-4周 |
| 对策拟定 | 2周 |
| 对策实施 | 6-8周 |
| 效果确认 | 3-4周 |
| 标准化 | 2周 |
| 检讨改进 | 1周 |

### 4.3 现状把握数据

**查检表设计**：
- 收集周期：通常2-4周
- 样本量：至少50-100例事件或观察
- 分层因素：时间、人员、环节、类型等

**柏拉图要求**：
- 项目数：5-8项
- 前2-3项累计≥80%
- 最后设"其他"项（<10%）

### 4.4 目标设定

**目标值计算公式**：
```
目标值 = 现状值 - (现状值 × 改善重点 × 圈能力)

改善重点 = 柏拉图前1-3项累计百分比（通常70-80%）
圈能力 = 圈员自评（通常60-80%）
```

**示例**（负向指标）：
- 现状值：跌倒率 0.6‰
- 改善重点：80%
- 圈能力：70%
- 目标值 = 0.6 - (0.6 × 0.8 × 0.7) = 0.6 - 0.336 = 0.264‰

### 4.5 鱼骨图（特性要因图）

**标准4M1E分类**：
- **人（Man）**：知识、态度、技能、人力
- **机（Machine）**：设备、工具、系统
- **料（Material）**：物资、药品、耗材
- **法（Method）**：流程、制度、标准
- **环（Environment）**：环境、布局、时间

**生成要求**：
- 每个大要因下3-5个中要因
- 每个中要因下1-3个小要因
- 总要因数：15-30个
- 圈选真因：3-6个

### 4.6 真因验证

**验证方法**：
- 查检验证（数据统计）
- 现场观察
- 问卷调查
- 访谈确认

**数据要求**：
- 每个待验证要因有对应验证数据
- 真因：验证支持率≥70%
- 非真因：验证支持率<50%

### 4.7 对策评价表

**评价维度**（每项1-5分）：
- 可行性
- 经济性
- 效益性
- 圈能力

**筛选标准**：
- 总分≥12分：采纳
- 总分9-11分：考虑
- 总分<9分：暂不采纳

### 4.8 效果确认

**必须包含**：
- 改善前后数据对比（相同收集方法）
- 折线图显示趋势
- 目标达成率计算
- 统计检验（卡方检验或t检验）

**目标达成率**：
```
达成率 = (改善前值 - 改善后值) / (改善前值 - 目标值) × 100%
```

**进步率**：
```
进步率 = (改善前值 - 改善后值) / 改善前值 × 100%
```

### 4.9 标准化文件

**SOP要素**：
- 目的
- 适用范围
- 职责分工
- 操作步骤（编号清晰）
- 注意事项
- 相关文件
- 修订记录

### 4.10 无形成果雷达图

**评价维度**（改善前后各评分1-5分）：
- QCC手法运用
- 团队凝聚力
- 沟通协调能力
- 责任感
- 解决问题能力
- 工作积极性
- 专业知识
- 品质意识

## Step 5: 文档生成

### Excel文件
使用openpyxl，参考 /mnt/skills/public/xlsx/SKILL.md

关键要求：
- 使用Excel公式计算（非硬编码）
- 柏拉图需含累计百分比公式
- 统计表需自动汇总

### Word文件
使用docx-js，参考 /mnt/skills/public/docx/SKILL.md

关键要求：
- 统一格式（字体、标题层级）
- 表格清晰
- 包含页眉页脚

### 图表
使用matplotlib/seaborn生成：
- 柏拉图（双Y轴）
- 折线图（趋势对比）
- 雷达图（无形成果）
- 甘特图
