# Medical Research AI - 医学科研全流程自动化

## 简介

模拟医学科研人员从 idea 到落实前的完整工作流程，将 6 个月的工作压缩到 10 分钟完成。

## 适用场景

当用户提到以下内容时使用此 skill：
- "科研想法"、"研究设计"、"文献调研"
- "有没有创新性"、"可行性如何"
- "帮我做个研究方案"、"写 protocol"
- "眼科科研"、"视光研究"
- "Idea 生成"、"gap 分析"

## 执行流程

### Step 1: 理解研究意图

向用户提问以明确：
- 具体的临床问题或研究方向
- 目标人群（如：儿童、老年人、特定疾病）
- 研究类型偏好（观察性/干预性/诊断性等）
- 可用资源（单中心/多中心，样本量限制）

### Step 2: PubMed 文献检索

使用 PubMed E-utilities API：

```bash
# 构建 MeSH + 关键词检索式
# 获取文献列表（目标 500-1000 篇）
# 提取摘要、MeSH 词、发表时间
```

### Step 3: 文献分析

- 研究类型统计
- 时间趋势分析
- 方法学总结
- 主要发现分类
- 研究质量评估

### Step 4: Gap 识别

基于文献分析，识别：
1. 研究人群空白
2. 研究设计空白
3. 测量指标空白
4. 随访时间空白
5. 干预措施空白

### Step 5: 方向生成

为每个 gap 生成对应研究方向，包括：
- 研究题目
- 研究设计
- 主要终点
- 创新性评分
- 可行性评分
- 预期贡献

### Step 6: 详细方案生成

对 Top 3 方向生成：
- 完整研究设计
- 样本量计算
- 测量工具建议
- 统计分析计划
- 时间安排
- 预算估算
- 风险评估

### Step 7: 输出材料

1. 研究方案文档
2. 文献综述
3. 创新性论证
4. Protocol 模板
5. 伦理申请材料
6. 知情同意书模板

## 注意事项

1. **PubMed API 无配额限制**，优先使用
2. **全文获取受限**，主要基于摘要分析
3. **创新性评估基于 Gap**，不保证真正突破
4. **可行性需人工审核**，不了解当地实际情况
5. **医学专业性**，眼科/视光领域优先

## 输出格式

使用 Markdown 格式，结构化输出：
- 文献分析数据用表格
- 研究方向用评分排序
- 方案用标准模板
- 关键决策点用标注
